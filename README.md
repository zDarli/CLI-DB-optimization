Приложение-CLI для справочника сотрудников. Выполняет создание таблиц, ввод и генерацию записей, пакетную загрузку (1 000 000 строк), выборки и бенчмарки, а также оптимизацию для ускорения запроса п.5.

## Требования
- Python ? 3.10
- SQLite (стандартный модуль `sqlite3`)
- `faker`, `pytils`/`transliterate` — для генерации данных

## Запуск
через python cli.py N args, где N - номер режима работы, args - аргументы

Пояснения:
cli.py - файл, реализирующий CLI(Command Line Interface) приложение, через argparse выбираются нужные таски для исполнения.
db.py - файл, реализирующий соединение с БД
models.py содержит SQL запрос для создания таблицы сотрудников и класс Employee, описывающий поведение объекта сотрудника
tasks/
-create.py - 1 таск
-insert.py - 2 таск
-list.py - 3 таск
-fill.py - 4 таск
-timed_list.py - 5/6 таск
Важные моменты:

В описании 2 пункта задания написано: "При генерации строчек в базу создавать объект и его отправлять в базу/формировать строчку для отправки нескольких строк в БД.
У объекта должны быть методы, которые:
- отправляют объект в БД,
- рассчитывают возраст (полных лет)." и в описании 3 пункта: "Вывести ФИО, Дату рождения, пол, кол-во полных лет.".
Отсюда можно сделать вывод что требуется в базе данных хранить полный возраст, что не практично, тк пройдет год и данные устареют.
Метод рассчета полных лет реализован, но закомментен. Вместо этого при выводе строк возраст считается отдельно.
